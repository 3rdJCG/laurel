## ADDED Requirements

### Requirement: Forge Today 画面にメール候補カードを1件ずつ表示する
mailtrack フォルダに未処理の JSON ファイルがある場合、Forge Today 画面は先頭の1件をカード形式で表示する SHALL。

#### Scenario: 未処理ファイルがある状態で Forge Today を開く
- **GIVEN** mailtrack フォルダに未処理の JSON ファイルが存在する
- **WHEN** ユーザーがサイドバーから Forge Today を選択する
- **THEN** 最初の未処理メールのカードが表示される
- **AND** カードには件名・送信者・受信日時・本文プレビュー（最大 200 文字）が表示される

#### Scenario: カードの本文プレビューは HTML タグを除去したテキストで表示する
- **WHEN** mailtrack ファイルの `body` フィールドが HTML 形式である
- **THEN** カードの本文プレビューは HTML タグを除去したプレーンテキストで表示される

### Requirement: カードをプロジェクトに割り当てられる
ユーザーはカードに表示されたメールをいずれかのプロジェクトのタスクとして登録できる SHALL。

#### Scenario: プロジェクトを選択して「追加」する
- **WHEN** ユーザーがドロップダウンからプロジェクトを選択して「追加」ボタンをクリックする
- **THEN** 選択したプロジェクトに新しいタスクが作成される
- **AND** タスクの `title` はメール件名になる
- **AND** タスクの `source` は `{ type: 'mailtrack', fileName, body }` になる
- **AND** 次の未処理カードが表示される（なければ処理済みメッセージを表示する）

#### Scenario: プロジェクト未選択で「追加」ボタンをクリックする
- **WHEN** ユーザーがプロジェクトを選択せずに「追加」ボタンをクリックする
- **THEN** タスクは作成されず、ドロップダウンが未選択であることを示す

### Requirement: カードを破棄できる
ユーザーはメールをタスクとして登録せずに処理済みにできる SHALL。

#### Scenario: 「破棄」ボタンをクリックする
- **WHEN** ユーザーがカードの「破棄」ボタンをクリックする
- **THEN** そのメールは処理済みとして記録される
- **AND** タスクは作成されない
- **AND** 次の未処理カードが表示される（なければ処理済みメッセージを表示する）

### Requirement: 全候補を処理済みにしたら完了メッセージを表示する
未処理のメールが0件になった場合、Forge Today 画面は完了メッセージを表示する SHALL。

#### Scenario: すべてのカードを処理した後
- **WHEN** 最後の未処理カードを割り当てまたは破棄する
- **THEN** 「受信メールはすべて処理済みです」メッセージが表示される

### Requirement: mailtrackDir 未設定時は設定誘導を表示する
mailtrack フォルダが設定されていない場合、Forge Today 画面は設定への誘導を表示する SHALL。

#### Scenario: mailtrackDir が未設定の状態で Forge Today を開く
- **WHEN** 設定で mailtrack フォルダが指定されていない
- **WHEN** ユーザーが Forge Today を開く
- **THEN** 「設定画面で mailtrack フォルダを指定してください」メッセージと設定画面へ移動するボタンが表示される
