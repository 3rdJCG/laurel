## ADDED Requirements

### Requirement: 同一階層内でタスクをドラッグ＆ドロップで並び替えられる
プロジェクト画面のタスク一覧で、同じ親を持つタスク間のドラッグ＆ドロップによる並び替えができる SHALL。

#### Scenario: タスクをドラッグして別の位置に移動できる
- **WHEN** ユーザーがタスクアイテムをドラッグして同一階層の別の位置にドロップする
- **THEN** タスクの表示順が変更され、新しい順序が即時反映される

#### Scenario: 並び替えた順序が保存される
- **WHEN** ユーザーがドラッグ＆ドロップで並び替えを完了する
- **THEN** 変更された表示順がデータとして保存され、再読み込み後も保持される

### Requirement: 異なる階層（親の異なる）タスク間の移動はできない
ドラッグ操作は同一の親を持つタスク間に限定される SHALL。異なる親への移動（親子関係の変更）はドロップ操作でブロックされる。

#### Scenario: 別の親を持つタスクへのドロップはキャンセルされる
- **WHEN** ユーザーがタスクを別の親タスクの子タスクエリアにドロップしようとする
- **THEN** ドロップ操作がキャンセルされ、タスクは元の位置に戻る

### Requirement: タスクデータに表示順フィールドを持つ
タスクデータは `order` フィールド（数値）を持ち、同一階層内での表示順を表す SHALL。

#### Scenario: タスク作成時に order フィールドが設定される
- **WHEN** 新しいタスクが作成される
- **THEN** `order` フィールドに同一階層の末尾となる値が設定される

#### Scenario: 並び替え後に order フィールドが更新される
- **WHEN** ドラッグ＆ドロップで並び替えが完了する
- **THEN** 移動したタスクおよび影響を受けた同階層タスクの `order` フィールドが更新される
