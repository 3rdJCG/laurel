## ADDED Requirements

### Requirement: 子タスクを持つタスクアイテムに展開・折りたたみトグルを表示する
子タスクが1件以上存在するタスクアイテムには展開・折りたたみトグルアイコンを表示する SHALL。子タスクが存在しないタスクアイテムにはトグルを表示しない。

#### Scenario: 子タスクを持つタスクにトグルが表示される
- **WHEN** 子タスクが1件以上存在するタスクアイテムがプロジェクト画面に表示される
- **THEN** タスクアイテムに展開・折りたたみトグルアイコンが表示される

#### Scenario: 子タスクを持たないタスクにはトグルが表示されない
- **WHEN** 子タスクが存在しないタスクアイテムがプロジェクト画面に表示される
- **THEN** 展開・折りたたみトグルアイコンは表示されない

### Requirement: トグルをクリックして子タスクの表示・非表示を切り替えられる
トグルをクリックするたびに子タスクの表示・非表示が切り替わる SHALL。

#### Scenario: 折りたたみ状態でトグルをクリックすると子タスクが表示される
- **WHEN** 折りたたみ状態のタスクアイテムのトグルをクリックする
- **THEN** そのタスクの直接の子タスクが一覧に展開表示される

#### Scenario: 展開状態でトグルをクリックすると子タスクが非表示になる
- **WHEN** 展開状態のタスクアイテムのトグルをクリックする
- **THEN** そのタスクの子タスク（および孫タスク以降）がすべて非表示になる

### Requirement: 初期状態はすべて折りたたみ状態である
プロジェクト画面を開いたとき、子タスクを持つすべてのタスクは折りたたみ状態で表示される SHALL。

#### Scenario: プロジェクト画面を開いたときの初期状態
- **WHEN** ユーザーがプロジェクト画面を開く
- **THEN** 子タスクを持つタスクはすべて折りたたみ状態となり、ルートタスクのみが表示される

### Requirement: 展開・折りたたみ状態はセッション中に保持される
同一プロジェクト画面を表示している間、展開・折りたたみ状態は操作した結果を保持する SHALL。プロジェクト画面から離脱すると初期状態にリセットされる。

#### Scenario: 展開操作後に状態が保持される
- **WHEN** ユーザーがタスクを展開した後、同プロジェクト画面内で別の操作を行う
- **THEN** 展開したタスクは引き続き展開状態を維持する

#### Scenario: 別プロジェクトに移動すると展開状態がリセットされる
- **WHEN** ユーザーが展開操作後に別のプロジェクト画面または Home 画面に移動する
- **THEN** 展開・折りたたみ状態が初期状態（すべて折りたたみ）にリセットされる
