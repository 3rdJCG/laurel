## Context

プロジェクト画面のタスク一覧は現在「平坦なリスト」として表示が定義されているが、タスクモデルは最大5層の親子階層を持つ。子タスクの展開・折りたたみがなければ深い階層で画面が埋まり、ドラッグ並び替えがなければ優先順位の調整ができない。

`define-task-operations-ui` でステータスバッジによるインライン変更は定義済みのため、本 change ではステータス変更UXは扱わない。

## Goals / Non-Goals

**Goals:**
- 子タスクを持つタスクアイテムに展開・折りたたみトグルを追加する
- 同一階層内でのドラッグ＆ドロップによる並び替えを実装する
- 展開状態をセッション中（ページ遷移なし）は保持する

**Non-Goals:**
- 展開状態の永続化（sessionStorage/localStorage への保存）
- 階層をまたいだ移動（親の変更）
- キーボードのみでの並び替え操作（アクセシビリティ向上は将来対応）

## Decisions

### 展開・折りたたみ：ローカル state で管理

展開状態は `Set<taskId>` または `Record<taskId, boolean>` としてプロジェクト画面コンポーネントの state に保持する。ページ遷移時にリセットされる（永続化しない）。

**初期状態**: 全タスクを折りたたみ状態にする（デフォルト展開だとタスク数が多い場合に一覧が長くなりすぎる）。

**代替案**: すべて展開状態をデフォルトにする案は、深い階層を持つプロジェクトで視認性が著しく低下するためリジェクト。

### 並び替え：`@dnd-kit/core` + `@dnd-kit/sortable` を採用

React 向けのドラッグ＆ドロップライブラリとして `@dnd-kit` を採用する。軽量でアクセシビリティ対応があり、Electron との相性も良い。

**スコープ**: 同一親を持つタスク間の並び替えのみ。異なる親への移動は対象外。

**表示順の保存**: タスクデータに `order: number` フィールドを追加し、並び替え後に `data:save-project` を呼ぶ。`order` の初期値は配列インデックスとする。

**代替案**: `react-beautiful-dnd` は React 18 の Strict Mode と相性が悪いためリジェクト。配列インデックスのみで管理する案は複数のタスクの `order` を同時に更新する必要があり実装が複雑になるため、明示的な `order` フィールドを採用。

## Risks / Trade-offs

- **`order` フィールド追加によるデータモデル変更**: 既存の task データに `order` フィールドがない場合、読み込み時にデフォルト値（配列インデックス）を補完する処理が必要。→ 読み込み時のマイグレーション処理で対応。
- **ドラッグ操作と折りたたみの競合**: 折りたたまれた親をドラッグしても子の順序は影響を受けない。→ 親単位の並び替えのみのため問題なし。
