## Context

`privacy-guarantee` スペックで「About 画面またはフッターでプライバシー情報と MIT ライセンスを表示する」という要件が存在する。現在の画面レイアウト定義（`define-screen-layout`）には About 画面が含まれていない。

## Goals / Non-Goals

**Goals:**
- About モーダルを実装し、アプリ名・バージョン・ライセンス・プライバシー情報を表示する
- ナビゲーションから About にアクセスできる導線を設ける
- `privacy-guarantee` スペックの要件を満たす

**Non-Goals:**
- ライセンス文の全文表示（「MIT License」の言及と外部リンクで十分）
- アップデート通知機能
- サードパーティライブラリのライセンス一覧

## Decisions

### 表示形式：モーダルダイアログ

About 画面は独立した画面遷移ではなく、現在の画面上に重ねるモーダルで実装する。内容が少なく（1画面に収まる量）、遷移コストを下げたい。

**導線**: ナビゲーションバーまたはサイドバーの「About」リンク・アイコンから開く。`settings-screen-ui` の設定モーダルと同様のパターン。

**代替案**: 専用のルート画面にする案は、コンテンツが少ないため過剰。フッターへの常時表示は画面スペースを無駄に占有するためリジェクト。

### バージョン情報の取得：IPC 経由で package.json から読む

`app.getVersion()` をメインプロセスで呼び出し、`app:get-version` IPC でレンダラーに渡す。ビルド時に埋め込む方式と比べて実装がシンプル。

**代替案**: electron-vite の `define` でビルド時に `__APP_VERSION__` を埋め込む方法もある。IPC 追加を避けたい場合はこちらが軽量。→ 今回は `settings:get` と同様のパターンで IPC 方式を採用し一貫性を保つ。

## Risks / Trade-offs

- **IPC 追加のオーバーヘッド**: `app:get-version` という IPC を1件追加するが、影響は最小限。
- **バージョン表示のズレ**: 開発環境では `package.json` のバージョンが表示されるが、リリースビルドでは Electron の `app.getVersion()` が使われるため、常に一致する。
