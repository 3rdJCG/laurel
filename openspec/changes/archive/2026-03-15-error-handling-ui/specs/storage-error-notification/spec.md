## ADDED Requirements

### Requirement: プロジェクト保存失敗時にエラーバナーを表示する
`data:save-project` IPC がエラーを返した場合、プロジェクト画面にエラーバナーを表示する SHALL。バナーは手動で閉じるまで表示され続ける。

#### Scenario: 保存失敗時にエラーバナーが表示される
- **WHEN** プロジェクトの保存操作（タスク作成・更新・削除）が失敗する
- **THEN** 画面上部にエラーバナーが表示され、保存に失敗した旨のメッセージが伝わる

#### Scenario: エラーバナーに再試行ボタンが表示される
- **WHEN** 保存失敗のエラーバナーが表示されている
- **THEN** バナー内に「再試行」ボタンが表示される

#### Scenario: 再試行ボタンをクリックすると保存を再実行する
- **WHEN** ユーザーがエラーバナーの「再試行」ボタンをクリックする
- **THEN** 失敗した保存操作が再実行される

#### Scenario: 再試行が成功するとバナーが消える
- **WHEN** 再試行の保存操作が成功する
- **THEN** エラーバナーが非表示になる

### Requirement: データ読み込み失敗時にエラーバナーを表示する
`data:load-all` IPC がエラーを返した場合、Home 画面またはアプリ起動時にエラーバナーを表示する SHALL。

#### Scenario: 全データ読み込み失敗時にエラーが通知される
- **WHEN** アプリ起動時の `data:load-all` が完全に失敗する（ディレクトリアクセス不可等）
- **THEN** エラーバナーが表示され、データを読み込めなかった旨と原因（パスが見つからない等）が表示される

### Requirement: 同種のエラーは集約して1件のバナーとして表示する
複数の保存失敗が連続した場合でも、同種のエラーバナーは1件に集約して表示する SHALL。

#### Scenario: 複数の保存失敗が集約される
- **WHEN** 短時間に複数のタスク保存が連続して失敗する
- **THEN** エラーバナーは1件のみ表示され、重複してスタックされない
