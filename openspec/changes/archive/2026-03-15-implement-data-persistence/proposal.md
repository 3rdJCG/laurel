## Why

現在 Laurel のデータは `useState` によるインメモリ管理のため、アプリ終了・シャットダウンですべて消える。タスク管理アプリとして実用するには、アプリ再起動後もデータが保持される永続化層が必要。また、チームや家族で同じファイルサーバ上のデータを共有したいユースケースに対応するため、保存先フォルダを自由に指定でき、かつ外部からの変更がリアルタイムで反映される設計が求められる。

## What Changes

- アプリ終了時に Project・Task データをローカルファイルに保存する
- アプリ起動時にファイルからデータを読み込む
- デフォルト保存先として OS 標準のユーザーデータフォルダを使用する
- ユーザーが任意のフォルダを保存先として指定できる設定機能を追加する
- 指定フォルダは Windows の UNC パス（`\\server\share\folder`）や Samba マウントパスも含む任意のパスを受け付ける
- 保存先のデータファイルを監視し、外部（別デバイス・別アプリ）による変更を検知してアプリに即時反映する
- 保存先設定自体は OS 標準フォルダに保存する（設定と実データを分離）

## Capabilities

### New Capabilities

- `data-storage`: Project・Task データの JSON ファイルへの読み書き
- `storage-location-setting`: 保存先フォルダの設定・変更（デフォルト or カスタムパス）
- `file-watch-sync`: データファイルの変更を監視し、外部変更をアプリに即時反映する

### Modified Capabilities

（なし）

## Impact

- `src/main/` に永続化層（ファイル読み書き・設定管理・ファイル監視）を追加
- `src/preload/` に IPC ブリッジを追加（レンダラー ↔ メインプロセス間のデータ受け渡し・変更通知）
- `src/renderer/` のデータ管理を `useState` インメモリから IPC 経由の永続化に移行
- 依存追加: `chokidar`（ファイル監視。ネットワーク共有でのポーリングモード対応）
- 依存: Node.js `fs` モジュール（Electron 標準）、`electron.app.getPath('userData')`
