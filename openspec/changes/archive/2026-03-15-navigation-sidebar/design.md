## Context

現在のナビゲーションは Home 画面内のプロジェクトリストをクリックしてプロジェクト画面に遷移し、戻るボタンで Home に戻る形式。この設計はプロジェクト数が増えると切り替えコストが高い。Settings・About も別途定義されたが導線がない。

`settings-screen-ui` は設定モーダルを定義済み。`about-screen-ui` は About モーダルを定義済み。どちらも「ナビゲーションから開く」とだけ書かれており、具体的な導線が未定義。

## Goals / Non-Goals

**Goals:**
- 左サイドバーに Home・全プロジェクト・Settings・About へのナビゲーション項目を配置する
- サイドバーをハンバーガーボタンで展開（ラベル付き）/ 折りたたみ（アイコンのみ）に切り替えられる
- 折りたたみ状態をローカルストレージに保存して再起動後も復元する
- サイドバー選択でアクティブな画面が切り替わる（Home / 各プロジェクト）

**Non-Goals:**
- サイドバー上でのプロジェクト並び替え（drag & drop）
- プロジェクトのグループ化・折りたたみ（多数プロジェクト対応は将来）
- モバイル・レスポンシブ対応

## Decisions

### サイドバーの構造

```
[ハンバーガーボタン]
─────────────────
🏠 Home
─────────────────
📁 Project A      ← プロジェクト一覧（ULID 作成順）
📁 Project B
📁 Project C
─────────────────
（下部固定）
⚙️ Settings       ← クリックで設定モーダルを開く
ℹ️ About          ← クリックで About モーダルを開く
```

展開時はアイコン + ラベル（幅 220px 程度）、折りたたみ時はアイコンのみ（幅 48px 程度）。

### 折りたたみ状態の管理：localStorage に保存

`localStorage.getItem('sidebar-collapsed')` で起動時に復元する。セッションをまたいで状態を保持したいため localStorage を選択。

**代替案**: `sessionStorage` はタブを閉じるたびにリセットされ、毎回開閉操作が必要になるためリジェクト。

### 画面状態管理：`currentView` を App.tsx の state として持つ

```ts
type View = { type: 'home' } | { type: 'project'; projectId: string }
```

Settings と About はモーダルとして既存定義のため `View` には含めず、`isSettingsOpen` / `isAboutOpen` を別 state として管理する。

**代替案**: React Router を導入する案はルーティングライブラリの追加が必要で、Electron の hash router 設定も必要になりオーバーキル。シンプルな state 管理で十分。

### Home 画面の役割変更

現在 Home 画面は「プロジェクト一覧カード + プロジェクト選択 → 遷移」の役割を持つ。サイドバー導入後もプロジェクト一覧は Home 画面にそのまま残す。サイドバーからも Home 画面からもプロジェクトへ遷移できる（二重導線）。

**補足**: 長期的には Home 画面を「ダッシュボード（直近タスク等）」に変えることも考えられるが、そのスコープは本 change 外。

## Risks / Trade-offs

- **プロジェクト数が多い場合のサイドバーの縦スクロール**: プロジェクトリストは縦スクロール可能にする。100件超は想定外のため無限スクロールは不要。
- **Home 画面の役割の重複**: サイドバーにプロジェクト一覧があり、Home 画面にもある。→ 今フェーズでは冗長を許容し、将来の Home 画面リデザインで整理する。
- **`about-screen-ui` が「ナビゲーションバー」に About を置くと定義している**: サイドバーの下部フッターに About を配置することで要件を満たす（ナビゲーション = サイドバーと解釈）。
