## Context

Laurel はローカルファースト設計の Electron アプリ。GitHub 等でのオープンソース公開に向け、ライセンス・プライバシーポリシー・README などの公開必須ドキュメントが未整備の状態。また「外部に情報を送信しない」という設計方針はコードレベルで担保されていない。

## Goals / Non-Goals

**Goals:**
- MIT ライセンスを明示し、商用利用を含む自由な利用を許可する
- プライバシーポリシーを文書化し、外部送信がないことを保証する
- README でプロジェクトの目的・使い方・開発方法を説明する
- Electron の CSP 設定により、外部ネットワーク通信をコードレベルで禁止する
- アプリ内 UI でライセンス・プライバシー情報をユーザーに提示する

**Non-Goals:**
- 翻訳（英語/日本語 どちらか一方で可）
- サードパーティ監査・法務レビュー
- CLA（Contributor License Agreement）の整備

## Decisions

### 1. ライセンス: MIT を採用

**理由:** 商用利用・改変・再配布を最大限許可し、最もシンプルな条件で自由な利用を保証できる。GPL 系だとアプリに組み込む際の制約が生じるため不採用。

### 2. CSP (Content Security Policy) で外部通信を制限

**理由:** `default-src 'self'` + `connect-src 'none'` を Electron の `webPreferences` と `session.defaultSession.webRequest` で設定することで、レンダラーからの外部 HTTP リクエストをコードレベルで遮断する。ドキュメントだけでなく技術的保証を提供する。

**代替案:** `net.setOnline(false)` や `protocol` ハンドラーによる遮断も検討したが、CSP が最も標準的で侵襲性が低い。

**将来の外部通信機能への対応:**
CSP は「許可するオリジンのホワイトリスト」として機能するため、将来的に外部通信が必要な機能（例: カレンダー同期・Webhook 通知）を追加する場合でも、設定を一元管理する `csp.ts` モジュールを用意することで拡張可能。

```
// 例: 将来の拡張イメージ
const CSP_CONNECT_SRC = process.env.ENABLE_SYNC
  ? "'self' https://api.example.com"
  : "'none'";
```

デフォルトは `'none'`（外部通信なし）を維持しつつ、ユーザーが明示的に機能を有効化した場合のみ許可オリジンを追加する設計にする。これにより「デフォルトで安全、必要に応じて拡張可能」な構造が保てる。

### 3. プライバシーポリシーは `docs/PRIVACY.md` に配置

**理由:** Markdown で管理することでコードと同じリポジトリで追跡・レビューが可能。アプリ内 About 画面からリンクまたは内容を表示する。

### 4. アプリ内 About 画面を新設

**理由:** ユーザーがアプリを使用中にライセンス・プライバシー情報へアクセスできるようにする。外部リンクを開かずにアプリ内で完結させる（ローカルファーストの一貫性）。

## Risks / Trade-offs

- [CSP 設定が開発時に影響する可能性] → 開発環境と本番環境で設定を分岐させる（`isDev` フラグ使用）
- [将来の外部通信機能追加時に CSP 設定変更が必要] → `csp.ts` に集約し、機能フラグで制御する設計にすることで局所的な変更で対応可能
- [README の陳腐化] → CLAUDE.md の開発方針と README の内容を一致させることで最小化

## Migration Plan

1. `LICENSE` ファイル追加（既存コードに影響なし）
2. `README.md` 追加（既存コードに影響なし）
3. `docs/PRIVACY.md` 追加
4. `src/main/csp.ts` モジュール作成 + Electron メインプロセスに CSP 設定を追加（動作確認必須）
5. About コンポーネントをレンダラーに追加
6. 動作確認: `npm run dev` で外部通信が遮断されることを確認

ロールバック: ドキュメント系は削除するだけ。CSP 設定はコメントアウトで即時無効化可能。
