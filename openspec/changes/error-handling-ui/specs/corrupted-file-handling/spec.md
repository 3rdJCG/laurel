## ADDED Requirements

### Requirement: 破損JSONファイルを検出してスキップし、他のプロジェクトは正常に読み込む
`data:load-all` 時に JSON parse が失敗したファイルはスキップし、残りのプロジェクトファイルは読み込みを継続する SHALL。

#### Scenario: 1ファイルが破損していても他のプロジェクトは表示される
- **WHEN** 保存先ディレクトリに1つの破損した JSON ファイルが存在する状態でアプリを起動する
- **THEN** 破損ファイルはスキップされ、正常な他のプロジェクトファイルは読み込まれて表示される

### Requirement: 破損ファイルを検出した場合に警告バナーを表示する
スキップしたファイルが存在する場合、Warning バナーで件数とファイルパスをユーザーに通知する SHALL。

#### Scenario: 破損ファイルスキップ時に Warning バナーが表示される
- **WHEN** 1つ以上のプロジェクトファイルが parse エラーでスキップされた状態でアプリが起動する
- **THEN** スキップしたファイル数とファイルパスを示す Warning バナーが表示される

#### Scenario: 破損ファイルの場所をファイルマネージャーで開けるボタンが表示される
- **WHEN** 破損ファイルの Warning バナーが表示されている
- **THEN** 「ファイルを開く」ボタンが表示され、クリックするとシステムのファイルマネージャーで対象ファイルの場所が開く

### Requirement: バージョン不整合ファイルも破損ファイルと同様に扱う
プロジェクトファイルの `version` フィールドが未対応の値の場合、そのファイルをスキップし Warning バナーで通知する SHALL。

#### Scenario: 未対応バージョンのファイルがスキップされる
- **WHEN** `version` フィールドが現在のアプリがサポートしていない値のプロジェクトファイルが存在する
- **THEN** そのファイルはスキップされ、Warning バナーで通知される
