## ADDED Requirements

### Requirement: Project 画面からルートタスクを追加できる
Project 画面のタスク一覧末尾に「＋ タスクを追加」ボタンが表示され、押下するとインラインのタイトル入力フォームが展開される SHALL。

#### Scenario: タスク追加ボタンを押下するとフォームが表示される
- **WHEN** ユーザーが Project 画面の「＋ タスクを追加」ボタンを押下する
- **THEN** タスク一覧の末尾にタイトル入力フォームが展開される

#### Scenario: タイトルを入力して確定するとルートタスクが追加される
- **WHEN** ユーザーがタイトルを入力して Enter キーを押す、または確定ボタンを押す
- **THEN** `parentId` を持たないルートタスクが作成され、タスク一覧の末尾に表示される

#### Scenario: Escape キーでフォームがキャンセルされる
- **WHEN** ユーザーがフォーム表示中に Escape キーを押す
- **THEN** フォームが閉じられ、タスクは作成されない

#### Scenario: タイトルが空のまま確定しようとしても追加されない
- **WHEN** ユーザーがタイトルを入力せずに確定操作を行う
- **THEN** タスクは作成されず、フォームはそのまま表示され続ける

### Requirement: タスク一覧が空のとき空状態UIとタスク追加ボタンが表示される
Project 画面にタスクが1件も存在しない場合、タスクが存在しないことを示すメッセージと「＋ タスクを追加」ボタンを表示する SHALL。

#### Scenario: タスクが0件のときは空状態メッセージが表示される
- **WHEN** Project 画面を開いたときにそのプロジェクトにタスクが存在しない
- **THEN** 「タスクはまだありません」等の空状態メッセージと「＋ タスクを追加」ボタンが表示される

### Requirement: 作成されたルートタスクは永続化される
ルートタスクを追加した後、`data:save-project` IPC を通じてデータが保存される SHALL。

#### Scenario: タスク作成後にデータが保存される
- **WHEN** ユーザーがルートタスクの作成を確定する
- **THEN** プロジェクトデータが `data:save-project` で保存され、アプリを再起動しても追加したタスクが表示される
