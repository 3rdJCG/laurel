## ADDED Requirements

### Requirement: 外部変更をアプリに即時反映する
保存先のプロジェクトファイルが外部（別デバイス・別アプリ）によって変更・追加・削除された場合、1秒以内にアプリのデータに反映される SHALL。

#### Scenario: 別アプリでプロジェクトファイルを編集した場合に反映される
- **WHEN** アプリ起動中に別のエディタなどで `<project-ulid>.json` を編集・保存する
- **THEN** 1秒以内にアプリ上のプロジェクト・タスクデータが更新される

#### Scenario: 別クライアントがプロジェクトファイルを追加した場合に反映される
- **WHEN** 別のデバイスが保存先フォルダに新しい `<project-ulid>.json` を作成する
- **THEN** 1秒以内にアプリのプロジェクト一覧に新しいプロジェクトが追加される

#### Scenario: 別クライアントがプロジェクトファイルを削除した場合に反映される
- **WHEN** 別のデバイスが保存先フォルダから `<project-ulid>.json` を削除する
- **THEN** 1秒以内にアプリのプロジェクト一覧から該当プロジェクトが取り除かれる

### Requirement: 自己書き込みは反映トリガーにならない
アプリ自身の書き込みによって発生した change イベントは無視される SHALL。

#### Scenario: アプリがデータを保存しても再読み込みが発生しない
- **WHEN** ユーザーがタスクを追加し、アプリがファイルを書き込む
- **THEN** 書き込み直後の change イベントは無視され、余分な再読み込みは発生しない

### Requirement: ネットワーク共有上のファイルも監視できる
Samba / UNC パスに保存先を設定した場合も、ファイルの変更が検知される SHALL。

#### Scenario: Samba 共有フォルダ上のファイル変更が反映される
- **WHEN** 保存先が Samba 共有フォルダに設定されており、別デバイスがそのフォルダ内のファイルを更新する
- **THEN** アプリが変更を検知してデータを更新する
