## ADDED Requirements

### Requirement: デフォルト保存先は OS のドキュメントフォルダ内の laurel フォルダを使用する
保存先が未設定の場合、`app.getPath('documents')/laurel/` をデータ保存先とする SHALL。
Windows では `C:\Users\<user>\Documents\laurel\` に相当する。

#### Scenario: 初回起動時はドキュメントフォルダが使用される
- **WHEN** アプリを初めて起動する（設定ファイルが存在しない）
- **THEN** `<Documents>/laurel/` にデータが保存される

### Requirement: ユーザーは任意のフォルダを保存先として指定できる
設定画面からフォルダを選択し、保存先をカスタムパスに変更できる SHALL。UNC パス（`\\server\share\folder`）や Samba マウントパスを含む任意のパスを受け付ける。指定したフォルダに直接データが保存される（サブディレクトリは作成しない）。

#### Scenario: ローカルフォルダを保存先に指定する
- **WHEN** ユーザーが設定でローカルの任意フォルダを選択して保存先を変更する
- **THEN** 以降のデータが指定フォルダに直接保存される

#### Scenario: UNC パスを保存先に指定する
- **WHEN** ユーザーが `\\server\share\folder` 形式のパスを保存先に指定する
- **THEN** 指定パスに直接データが保存される

#### Scenario: アクセスできないパスを指定した場合エラーになる
- **WHEN** ユーザーがアクセス権限のないパスや存在しないパスを保存先に指定する
- **THEN** エラーが通知され、保存先はデフォルトパス（Documents/laurel）にフォールバックする

### Requirement: 保存先変更時に既存データを引き継ぐ
保存先を変更する際、変更前のデータを新しい保存先にコピーする SHALL。ただし新しい保存先に既存ファイルが存在する場合はそちらを使用する。

#### Scenario: 空のフォルダを保存先に変更する
- **WHEN** ユーザーがプロジェクトファイルを持たないフォルダを保存先に指定する
- **THEN** 現在の全プロジェクトファイルが新しい保存先にコピーされる

#### Scenario: 既存データがあるフォルダを保存先に変更する
- **WHEN** ユーザーが既にプロジェクト JSON ファイルが存在するフォルダを保存先に指定する
- **THEN** 既存ファイルが読み込まれ、移行前のデータは上書きされない

### Requirement: 設定ファイルは userData に保存される
保存先パスなどのアプリ設定は `<userData>/laurel/settings.json` に保存される SHALL。カスタム保存先に変更しても設定ファイルの場所は変わらない。

#### Scenario: 設定変更後にアプリを再起動しても設定が保持される
- **WHEN** カスタム保存先を設定してアプリを再起動する
- **THEN** 設定した保存先が復元され、カスタムフォルダのデータが読み込まれる
