## Context

`define-project-task-model` でタスクに `genre`（文字列 or null）と `tags`（文字列配列）フィールドが定義された。しかし現状ではジャンルは自由入力文字列に過ぎず、一覧管理・タグ補完・絞り込みの仕組みがない。

- ジャンルはプロジェクト横断の軸として使われる想定
- タグは各タスクに複数付与できるが、過去に使ったタグを再利用する手段がない
- プロジェクト画面ではすべてのタスクが一覧表示されるが、ジャンル/タグでの絞り込みができない

## Goals / Non-Goals

**Goals:**
- ジャンルを登録・一覧表示・削除できる管理UI
- タスク作成・編集フォームでのタグオートコンプリート
- プロジェクト画面でのジャンル・タグによるフィルタリング

**Non-Goals:**
- ジャンルの並べ替えや色付けなどのカスタマイズ
- タグの一括削除・リネーム（将来対応）
- ジャンル/タグ横断のグローバル検索
- プロジェクトをまたいだジャンル統計

## Decisions

### ジャンル管理：アプリ設定として永続化

ジャンル一覧を `settings.json`（`implement-data-persistence` で定義済み）の `genres` キーに文字列配列として保存する。

**代替案**: 各タスクの `genre` フィールドを集計して一覧を導出する案もあるが、削除済みジャンルの扱いや「登録したが未使用のジャンル」が表現できないためリジェクト。

**IPC**: `settings:get` / `settings:set` を流用する（追加 IPC なし）。

### ジャンル管理UI：設定画面（settings-screen-ui）内に配置

`settings-screen-ui` change で設定モーダルが定義済みのため、そのモーダルの一セクションとして「ジャンル管理」タブまたはセクションを追加する。

**代替案**: サイドバーに独立した管理パネルを置く案はナビゲーション複雑化を招くためリジェクト。

### タグサジェスト：全タスクから収集・メモリキャッシュ

プロジェクト内の全タスクの `tags` を結合・dedup してサジェスト候補とする。読み込み時にメモリキャッシュし、タスク保存時に更新する。

**代替案**: バックエンド IPC でタグ一覧を返す API を追加する案もあるが、既存の `project:get` でタスクデータは全取得済みのためフロントエンドで集計すれば済む。

### フィルターUI：プロジェクト画面内のフィルターバー

プロジェクト画面のタスク一覧上部にフィルターバーを配置。ジャンル（ドロップダウン）とタグ（マルチセレクト）でフィルタリングする。フィルター状態はコンポーネントのローカル state に持つ（URL や設定には保存しない）。

## Risks / Trade-offs

- **settings.json にジャンルを保存 → データファイルとの分離**: カスタム保存先にデータを移しても、ジャンル一覧は `userData` に残る。ジャンルをプロジェクトデータとして扱いたい将来の要求と競合する可能性あり。→ 今回はスコープ外とし、将来のリファクタリング課題として記録する。
- **タグ候補の収集コスト**: タスク数が多い場合に全タスクスキャンが重くなりうる。→ 初期実装ではプロジェクト単位でのスキャンとし、アプリ全体のタスク数が少ないフェーズでは問題ない。
- **ジャンル削除時の既存タスクの扱い**: ジャンルを削除してもタスクの `genre` フィールドは変わらない（文字列のままオーファン化する）。→ 削除時に「このジャンルを持つタスクが N 件あります」と警告表示し、ユーザーが判断する。
