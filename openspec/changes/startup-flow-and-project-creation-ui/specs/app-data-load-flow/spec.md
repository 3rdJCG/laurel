## ADDED Requirements

### Requirement: アプリ起動時にレンダラーがデータを読み込む
アプリ起動時、レンダラーのデータ管理層が `data:load-all` IPC を invoke し、全プロジェクト・タスクデータを取得する SHALL。この呼び出しはマウント後に1回だけ行われる。

#### Scenario: 起動時にデータが正常に読み込まれる
- **WHEN** アプリが起動し、レンダラーが初期化される
- **THEN** `data:load-all` が invoke され、保存済みのプロジェクト・タスクが画面に表示される

#### Scenario: データが存在しない状態で起動する
- **WHEN** 保存データが1件もない状態でアプリが起動する
- **THEN** `data:load-all` が invoke され、プロジェクト一覧が空の状態で表示される

### Requirement: データ読み込み中はローディング状態を表示する
`data:load-all` の応答が返るまでの間、ローディング状態を示す UI を表示する SHALL。通常の画面コンテンツはローディング完了後にのみ表示される。

#### Scenario: 読み込み中はローディング表示になる
- **WHEN** アプリが起動し `data:load-all` の応答を待っている
- **THEN** ローディングインジケーターが表示され、プロジェクト一覧などのコンテンツは表示されない

#### Scenario: 読み込み完了後に通常画面が表示される
- **WHEN** `data:load-all` の応答が正常に返る
- **THEN** ローディング表示が消え、通常の画面（Home 画面など）が表示される

### Requirement: データ読み込みに失敗した場合はエラー状態を表示する
`data:load-all` が失敗した場合、エラーメッセージを表示する SHALL。通常の画面コンテンツはエラー状態では表示されない。

#### Scenario: 読み込みエラー時にエラー表示になる
- **WHEN** `data:load-all` が例外またはエラー応答を返す
- **THEN** エラーメッセージが表示され、プロジェクト一覧などのコンテンツは表示されない
