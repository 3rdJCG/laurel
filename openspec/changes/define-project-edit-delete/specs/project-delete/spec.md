## ADDED Requirements

### Requirement: プロジェクトを削除できる
Home 画面のプロジェクトカードから削除操作を実行できる SHALL。誤削除を防ぐために確認ステップが必要である。

#### Scenario: 削除ボタン押下で確認UIが表示される
- **WHEN** ユーザーがプロジェクトカードの削除ボタン（ゴミ箱アイコン）を押下する
- **THEN** カード内にインライン確認UI（確認メッセージ・「削除」ボタン・「キャンセル」ボタン）が表示される

#### Scenario: 確認UIにタスク件数が表示される
- **WHEN** 削除確認UIが表示される
- **THEN** そのプロジェクトに属するタスクの件数が確認メッセージ内に表示される

#### Scenario: 削除を確認するとプロジェクトが消える
- **WHEN** ユーザーが確認UIで「削除」を選択する
- **THEN** プロジェクトが削除され、Home 画面の一覧から消える

#### Scenario: キャンセルするとプロジェクトは削除されない
- **WHEN** ユーザーが確認UIで「キャンセル」を選択する
- **THEN** プロジェクトは削除されず、確認UIが閉じてカード通常表示に戻る

### Requirement: プロジェクト削除時に紐付くタスクも連鎖削除される
プロジェクトを削除した場合、そのプロジェクトに属するすべてのタスク（サブタスク含む）も削除される SHALL。

#### Scenario: タスクが存在するプロジェクトを削除するとタスクも消える
- **WHEN** ユーザーがタスクを持つプロジェクトの削除を確認する
- **THEN** プロジェクトおよびそのプロジェクトに属するすべてのタスクが削除される

#### Scenario: タスクが0件のプロジェクトも削除できる
- **WHEN** ユーザーがタスクを持たないプロジェクトの削除を確認する
- **THEN** プロジェクトが削除され、一覧から消える
